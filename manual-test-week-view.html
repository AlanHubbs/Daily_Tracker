📋 5-Day Week View - Manual Testing Checklist

Note: Open index.html in a new tab to perform these tests.

Test 1: Visual Layout PENDING

  • ☐ Verify 5 columns are displayed (Monday through Friday)
  • ☐ Each column has a day name and date
  • ☐ Week navigation buttons (< and >) are visible in the header
  • ☐ Week date range is displayed (e.g., "Aug 18 - Aug 22")
  • ☐ Columns have equal width and proper spacing

Test 2: Adding Tasks PENDING

  • ☐ Add a new task using the input field
  • ☐ Verify the task appears in today's column
  • ☐ Add multiple tasks with different priorities
  • ☐ Verify priority colors are displayed correctly

Test 3: Week Navigation PENDING

  • ☐ Click the next week button (>)
  • ☐ Verify the week date range updates
  • ☐ Verify columns show the new week's dates
  • ☐ Click the previous week button (<)
  • ☐ Verify you return to the current week
  • ☐ Navigate multiple weeks forward and back

Test 4: Drag and Drop PENDING

  • ☐ Create a task for today
  • ☐ Drag the task to tomorrow's column
  • ☐ Verify the task moves to the new day
  • ☐ Refresh the page and verify the task stays in the new day
  • ☐ Try dragging multiple tasks between different days

Test 5: Data Persistence PENDING

  • ☐ Add tasks to different days of the week
  • ☐ Refresh the browser (F5 or Cmd+R)
  • ☐ Verify all tasks remain in their assigned days
  • ☐ Navigate to next week, add a task, then refresh
  • ☐ Verify the future task is saved

Test 6: Integration with Existing Features PENDING

  • ☐ Complete a task and verify XP/streak updates
  • ☐ Click a task title to edit it inline
  • ☐ Add notes to a task (📝 button)
  • ☐ Delete a task and verify it's removed
  • ☐ Use templates to create tasks
  • ☐ Verify recurring tasks appear in correct days
  • ☐ Export data and verify week view tasks are included

Test 7: Responsive Design PENDING

  • ☐ Resize browser window to mobile width (< 900px)
  • ☐ Verify columns stack vertically
  • ☐ Verify all functionality still works on mobile view
  • ☐ Return to desktop size and verify layout restores

Test 8: Edge Cases PENDING

  • ☐ Navigate to a week in the past
  • ☐ Add a task to a past date
  • ☐ Navigate far into the future (several months)
  • ☐ Add many tasks to a single day (10+)
  • ☐ Test with very long task titles

🎯 Expected Results

If all tests pass, you should have verified:

  • ✅ Week view displays correctly with 5 columns
  • ✅ Tasks can be created and appear in the correct day
  • ✅ Week navigation works forward and backward
  • ✅ Drag and drop moves tasks between days
  • ✅ All data persists after page refresh
  • ✅ Integration with existing features works
  • ✅ Responsive design adapts to screen size
  • ✅ Edge cases are handled gracefully
